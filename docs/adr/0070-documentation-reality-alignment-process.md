# 0070 - Documentation-Reality Alignment Process

## Status

Accepted

## Context

README and product language currently outrun a few shipped behaviors.

## Decision

Every user-facing claim must either be:

- directly tested
- demonstrated in the app
- documented as a limitation

Phase 3 updates README and audit documents as implementation lands.

## Consequences

Docs become a contract that must track reality, not a wishlist.

## Alternatives Considered

Leaving aspirational product language in place was rejected because it undermines trust during the stranger test.
